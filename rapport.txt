Correctifs CSS prioritaires – imports, syntaxe invalide, duplications

Objectif
- Corriger les erreurs de parsing, supprimer les duplications conflictuelles et normaliser les imports pour fiabiliser l’affichage et faciliter la maintenance.

1) Imports Google Fonts
Problème
- @import est présent au milieu du fichier (et même dans un bloc), ce qui est invalide/risqué. En CSS, @import doit être en tête de fichier, avant toute règle.

Action
- Déplacer tous les @import en tout début de style.css (et supprimer les doublons).
- Idéal: remplacer totalement @import par un <link> dans <head> avec font-display: swap. Si vous gardez @import, appliquez le bloc ci-dessous.

À mettre tout en haut de assets/portfolio/style.css (ligne 1):
@import url("https://fonts.googleapis.com/css2?family=Share+Tech+Mono&display=swap");

À supprimer dans le fichier:
- Toute autre ligne @import, notamment celle autour de ~ligne 921 et toute @import à l’intérieur d’un bloc.

2) Règle imbriquée invalide (syntaxe type SCSS non compilée)
Problème
- .days-left { ... .top-banner__title { ... } } vers ~lignes 607–636.

3) Duplications et conflits – .terminal-output

- Conserver UNE seule définition “canonique” pour .terminal-output.
- Utiliser .terminal-output.project-description pour les variations spécifiques (déjà présente, OK).
