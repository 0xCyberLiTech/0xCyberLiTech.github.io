// sécurité — audit 2026-03-22

$ cat audit-results.txt

✅ Anti-freeze — If-Modified-Since (304 si fichier inchangé = 0 parsing DOM) + requestIdleCallback (render pendant créneau idle navigateur). Polling proto-live.json également idle-schedulé.
✅ Accès LAN uniquement — Le dashboard SOC est exposé exclusivement sur le réseau local. Aucune exposition internet directe. Accessible via le reverse proxy authentifié.
✅ Intégration JARVIS optionnelle — Le SOC reste 100% fonctionnel sans JARVIS. Les actions proactives (ban-ip, restart-service) nécessitent une confirmation LLM avant exécution.
✅ Serveur durci — ModSecurity WAF activé, CrowdSec analyse comportementale, fail2ban jails SSH/HTTP, PasswordAuthentication désactivé, headers sécurité Apache/nginx.
✅ Rapports automatiques — soc-daily-report.py génère un rapport quotidien. Alertes email configurables. Snapshots UFW horodatés pour historique de la posture sécurité.
